# Наследование и полиморфизм в C#

В этом модуле раскрываются два фундаментальных принципа ООП — наследование и полиморфизм. Представлены базовые и производные классы, интерфейсы, абстракция, переопределение методов.

## Теория наследования

Наследование позволяет создавать новые классы на основе существующих, расширяя или изменяя их функционал.

```csharp
public class Animal
{
    public string Name;
    public virtual void Speak() {
        Console.WriteLine("Животное издаёт звук");
    }
}

public class Dog : Animal
{
    public override void Speak() {
        Console.WriteLine("Гав-гав!");
    }
}

Animal a = new Dog();
a.Name = "Шарик";
a.Speak(); // Гав-гав!
```

## Абстракция и виртуальные методы

Абстрактный класс описывает базовую структуру, но не реализует часть методов:

```csharp
public abstract class Shape {
    public abstract double Area();
}
public class Rectangle : Shape {
    public double Width, Height;
    public override double Area() {
        return Width * Height;
    }
}
```

## Интерфейсы

Интерфейс — это набор методов и свойств без реализации, служит контрактом для классов.

```csharp
public interface IMovable {
    void Move();
}
public class Car : IMovable {
    public void Move() {
        Console.WriteLine("Машина едет");
    }
}
```

## Полиморфизм

Полиморфизм позволяет работать с объектами разных классов через базовый интерфейс, переопределяя поведение методов.

- Компиляционный (перегрузка методов)
- Исполнительный (виртуальные/абстрактные методы, интерфейсы)

**Примеры:**
```csharp
List<Shape> figures = new List<Shape>();
figures.Add(new Rectangle { Width = 3, Height = 5 });
foreach (Shape s in figures) {
    Console.WriteLine(s.Area()); // для каждого типа вызывается своё определение
}
```

## Практические рекомендации
- Используйте наследование для логического расширения . Не допускайте чрезмерно глубоких иерархий.
- Абстрагируйте общий функционал в базовом классе или через интерфейс.
- Переопределяйте виртуальные методы для гибкости.
