# Классы и объекты в C#

В этом модуле последовательно изучаются основы объектно-ориентированного программирования (ООП) на C#: создание классов и объектов, свойства, методы, инкапсуляция и статические члены.

## Основы классов

Класс — это шаблон структуры данных, описывающий совокупность свойств и методов.

```csharp
public class Person {
    public string Name;    // Свойство
    public int Age;        // Свойство

    public void SayHello() // Метод
    {
        Console.WriteLine($"Привет, меня зовут {Name}");
    }
}
```

## Создание объектов

Объект создаётся как экземпляр класса и обладает собственными значениями свойств.

```csharp
Person p = new Person();
p.Name = "Виктор";
p.Age = 23;
p.SayHello();
```

## Конструкторы

Конструктор — специальный метод для инициализации объекта.

```csharp
public class Car {
    public string Model;
    public int Year;
    public Car(string model, int year) {
        Model = model;
        Year = year;
    }
}
Car car1 = new Car("Toyota", 2010);
```

## Модификаторы доступа

- `public` — открыт для всего кода
- `private` — доступен только внутри класса
- `protected` — доступен в наследуемых классах
- `internal` — внутри одной сборки

**Пример закрытого поля:**
```csharp
public class BankAccount {
    private double _balance;
    public void Deposit(double amount) {
        _balance += amount;
    }
    public double GetBalance() {
        return _balance;
    }
}
```

## Статические члены

Статические свойства и методы принадлежат классу в целом, а не конкретному объекту:
```csharp
public class MathTools {
    public static double Pi = 3.1415;
    public static int Square(int x) { return x * x; }
}
double c = MathTools.Pi;
int s = MathTools.Square(8);
```

## Деструкторы

Деструктор — метод, который вызывается при удалении объекта из памяти.
```csharp
class Demo {
    ~Demo() {
        Console.WriteLine("Объект уничтожен");
    }
}
```

## Практические рекомендации
- Соблюдайте принцип инкапсуляции: данные должны быть доступны только тем методам, которые действительно их используют.
- Конструкторы позволяют создавать максимально корректно инициализированные объекты.
- Грамотно используйте модификаторы доступа для безопасности и структурирования кода.

3. Создайте класс Account с инкапсулированными полями и открытыми методами Deposit() и GetBalance().
